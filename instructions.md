# Техническое задание: Custom DayZ Launcher (Fluent Design)

## 1. Стек технологий
- **Framework:** Tauri (Rust backend + React frontend)
- **Styling:** Tailwind CSS + Fluent UI design system (glassmorphism, subtle shadows, rounded corners)
- **Icons:** Lucide-React или FontAwesome
- **Steam Integration:** `steamworks-rs` (Rust) или `steamworks.js` (Node.js)
- **Server Query:** `gamedig` (библиотека для опроса игровых серверов)
- **State Management:** Zustand или React Context

## 2. Структура интерфейса
### Сайдбар (Sidebar) — виден всегда
- **Навигация:** Кнопки "Серверы", "Моды", "Настройки".
- **Поле ввода:** "Ник на сервере" (сохраняется в локальное хранилище).
- **Блок профиля Steam (внизу):** - Аватар пользователя (круглый).
  - Никнейм Steam.
  - SteamID64 (мелким шрифтом).

### Страница "Серверы" (Главная)
- Список серверов, формируемый на основе JSON из внешнего репозитория GitHub.
- **Карточка сервера должна содержать:**
  - Название сервера, Название карты.
  - Количество игроков (например, 45/60).
  - Пинг (обновляется в реальном времени через опрос).
  - Кнопка "Подключиться".

### Страница "Моды"
- Список всех модов, необходимых для серверов из списка.
- **У каждого мода:** Кнопка "Подписаться/Отписаться" и "Обновить" (вызывает проверку целостности через Steam API).
- **Групповые действия (сверху):** "Подписаться на всё", "Отписаться от всего", "Обновить всё".

### Страница "Настройки"
- **Путь к DayZ:** Поле с кнопкой выбора папки (автоматический поиск через реестр Windows: `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\Steam App 221100`).
- **Параметры запуска:** Текстовое поле для кастомных аргументов (например, `-maxMem=...`).
- **Тема:** Переключатель (Светлая / Темная / Системная).

## 3. Логика работы (Backend & Frontend)

### Этап 1: Инициализация и UI
- Настройка Tauri проекта.
- Реализация Fluent Design: использование эффектов акрила/стекла (glassmorphism), шрифтов Segoe UI / Inter.
- Создание адаптивного Layout с фиксированным сайдбаром.

### Этап 2: Steam API & Авторизация
- Интеграция Steamworks.
- При запуске лаунчер должен получать данные текущего пользователя Steam.
- Если Steam не запущен — выводить предупреждение.

### Этап 3: Работа с данными серверов
- Реализовать fetch-запрос к GitHub для получения JSON (формат: `[{ "ip": "...", "port": ... }]`).
- Написать Rust-команду для Tauri, которая использует `gamedig` для опроса каждого IP из списка и возвращает полные данные (название, игроки, моды).

### Этап 4: Система управления модами
- Реализовать логику подписки/отписки через `steamworks.ugc`.
- Реализовать функцию `ForceUpdateMod`: использование `DownloadItem(id, true)` для немедленной проверки обновлений.

### Этап 5: Запуск игры
- Сборка команды запуска:
  `DayZ_BE.exe -connect={IP} -port={PORT} -name={UserNick} "-mod={ModList}" {CustomArgs}`
- ModList должен формироваться автоматически на основе модов, которые требует конкретный сервер.

## 4. Специфические инструкции для Агента
- **Безопасность:** Не хардкодить пути, использовать переменные окружения.
- **Ошибки:** Если игра не найдена или Steam недоступен, лаунчер должен уведомлять пользователя через красивые Toast-нотификации.
- **Производительность:** Опрос серверов (ping/иглики) делать асинхронно, чтобы UI не зависал.
